UNITE

What Is UNITE?

Fig 1 from Koch et al. (2025) — UNITE takes posed images and outputs a semantically rich 3D point cloud.

Takes a set of RGB images → reconstructs the full 3D scene → labels every 3D point with what it is, which object it belongs to, and how it moves.

Key Properties

  • Works from RGB only — no depth sensor needed
  • Single forward pass, a few seconds per scene
  • Outputs are language-queryable (CLIP-aligned)
  • Instance-level segmentation in 3D

Per-Point Output

  • 3D coordinates — geometry
  • CLIP features — open-vocabulary semantics
  • Instance ID — object grouping
  • Articulation vector — how parts move

UNITE

How It Works

Fig 2 from Koch et al. (2025) — Architecture overview

1. Input

Set of RGB images from different viewpoints of the same scene

2. Backbone (VGGT)

Pre-trained vision transformer encodes images, fuses across views via attention → predicts 3D geometry

3. Semantic Heads (DPT)

Task-specific DPT heads on shared backbone predict semantics, instances, and articulation per point

4. Output

Dense 3D point cloud where each point has: coordinates, CLIP features, instance ID, articulation vector

Landscape

The Research Gap

Line A: VLAs with 3D

Several recent works add 3D to VLAs, but all use simple or implicit 3D:

  • SpatialVLA — 3D position encoding only (RSS 2025)
  • Spatial Forcing — implicit alignment, no 3D at inference (ICLR 2026)
  • 3D-VLA — point clouds but no semantics per point (ICML 2024)

Line B: Rich 3D Understanding

Foundation models for dense 3D scene understanding exist, but not connected to action:

  • UNITE — full semantic 3D from RGB (Koch et al. 2025)
  • VGGT — geometry backbone (CVPR 2025 Best Paper)

The Gap

No one has connected rich, dense 3D semantics to VLA action output.

  • Existing VLA+3D = sparse or geometry-only
  • UNITE's per-point semantics + instances never used for action
  • Room-scale navigation especially underserved
Proposed contribution: First to use UNITE-style dense semantic 3D features (CLIP + instance + articulation) as structured input to a VLA for object navigation.

Benchmark

Habitat ObjectNav Challenge

The Task

Agent is spawned in an unseen apartment. Given a target object category (e.g., "chair"), it must navigate to any instance of that category using only egocentric RGB-D and a compass.

HM3D Dataset

  • 1,000 real-world 3D scans of homes
  • Photorealistic rendering in Habitat simulator
  • 216 object categories with semantic annotations
  • Standard benchmark since 2023 Habitat Challenge

Key Metric: SPL

Rewards both finding the target and doing so efficiently. Standardizes path length by the shortest possible path.

Habitat ObjectNav — agent navigates to target object in photorealistic 3D scan
Why this benchmark: Room-scale navigation in real 3D scans. Agent must understand spatial layout, object identity, and efficient paths — exactly where 3D semantics should help.

Honest Assessment

Open Questions & Risks

UNITE Access

No public code or weights (Dec 2025 paper). Must ask Ropinski for access.

Fallback: VGGT (public, geometry-only) + add own semantic heads, or ConceptGraphs pipeline.

Multi-View Problem

UNITE expects multiple views of a scene. Navigating agent only sees one frame at a time.

Idea: Frame accumulation — collect frames as agent moves, progressively build 3D. But: latency, memory, quality?

Sim Quality

UNITE trained on real images. Habitat renders are synthetic. Unknown how well UNITE generalizes to sim.

Mitigation: HM3D uses photorealistic scans, not procedural — may be close enough.

Tokenization

How to compress a dense 3D point cloud into a fixed number of tokens for an LLM? No established method. Need to design and validate a strategy (FPS, voxel, learned projection).

Competitive Field

VLA+3D is a hot area — new papers weekly. Risk of being scooped. But: nobody has used UNITE-style dense features yet, and room-scale nav is underserved vs tabletop.
