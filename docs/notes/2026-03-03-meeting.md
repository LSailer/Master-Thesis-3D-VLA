## Pivot: Diffusion Policy → World Model

Previous approach: 3D diffusion policy / flow matching action heads (see `docs/specs/action-heads-spec.md`).

**New direction:** replace policy diffusion with a **Dreamer-style world model** that integrates agent dynamics.

## Core Idea

- Encoder stack produces latent representations from observations (RGB, depth, VGGT 3D features)
- World model learns environment dynamics in latent space
- After N real steps, the model **dreams** (imagines) future trajectories
- Dreamed trajectories are compared against **actual 3D observations** as ground truth
- Actor-critic trained on imagined rollouts (as in DreamerV3)

## Research Question

> Is a 3D observation prior helpful for a model-based (world model) approach?

Sub-questions:
- Does 3D semantic grounding (VGGT features) improve world model predictions?
- Can we analyze/interpret what the world model learns about 3D structure?
- Does dreaming with 3D priors lead to better ObjectNav performance vs 2D-only?

## Key Design Points

| Aspect | Decision |
|--------|----------|
| **Architecture** | Dreamer-style (RSSM or similar) |
| **Observation encoding** | RGB + depth + VGGT 3D features |
| **Dynamics model** | Latent transition model integrating agent actions |
| **Dreaming** | Imagine N-step rollouts, compare w/ real 3D observations |
| **Ground truth signal** | Actual 3D point observations from environment |
| **Analysis** | Probe what world model learns about 3D scene structure |

## Why World Model over Diffusion Policy

- Diffusion policy predicts action chunks but doesn't model environment dynamics
- World model explicitly learns how the world changes given actions
- Enables look-ahead planning via imagination
- Better sample efficiency (learn from imagined experience)
- Can directly evaluate whether 3D priors improve dynamics understanding
